# Добавление нового roll

Новый пост создается командой:

```sh
npm run new
```

Команда вызывает `add-new-roll.sh` без аргументов.

## Подготовка

1. Положить исходные JPG в `new-roll/photos/`.
2. Подготовить `new-roll/index.md`: заполнить поля, которые известны заранее.
3. Запустить `npm run new`.
4. Проверить созданный каталог `src/rolls/<next-number>/`.
5. При необходимости дозаполнить title, story, tags и описания фото.
6. Запустить `npm test` и `npm run build`.

## Что делает скрипт

`add-new-roll.sh`:

- находит максимальный числовой каталог в `src/rolls`;
- создает следующий номер;
- ставит дату публикации через `date +%F`;
- копирует front matter из `new-roll/index.md`;
- заменяет `number` и `date`;
- генерирует список `photos` по фактическим JPG в `new-roll/photos`;
- создает `xs-*` превью шириной 480px;
- создает рабочую версию изображения высотой до 1080px;
- останавливается при ошибках через `set -eu`.

Для тестов поддерживается переменная `ROLL_DATE`, чтобы дата была стабильной:

```sh
ROLL_DATE=2026-06-21 sh ./add-new-roll.sh
```

## Требования к изображениям

- Файлы должны быть JPG.
- Имена должны быть числовыми: `1.jpg`, `2.jpg`, `10.jpg`.
- Порядок сортируется через `sort -V`, поэтому `10.jpg` идет после `9.jpg`, а не после `1.jpg`.

## Защита от ошибок

Скрипт завершится с ошибкой, если:

- нет `new-roll/photos`;
- нет `new-roll/index.md`;
- в `new-roll/photos` нет JPG;
- целевой каталог уже существует.

Это сделано, чтобы не получить частично созданный или перезаписанный roll.
